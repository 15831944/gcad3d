Models use keycode  M







M int.Model Submodel Pos [Vec/Refsys]

Positioning an internal sub-model
  (all exported objects of the submodel (its joints) will be imported.
   Text of imported notes will be queried newly.)


Submodel
  - Select a sub-model from list (or use middle-mousebutton)
  - or select an existing sub-model
  
Position,Orientation
  Origin and orientation of the submodel

[Scale]:   magnifying scale; default is 1
 

Examples:
# Internal Submodel "SM1", rotated 45 degree around z-axis, scale=2
M20="SM1" R(P(100 0) 45) VAL(2)








M ext.Model Filename Pos [Vec/Refsys]

Positioning an external sub-model

Submodel
  - Select a sub-model from list (or use middle-mousebutton)
  - or select an existing sub-model

  Modelname: see Symbolic filenames.


Position,Orientation
  Origin and orientation of the submodel

[Scale]:   magnifying scale; default is 1
  

Example of an external model:
 M20 = "Data/Niet1.dat" R(P(100 0) DIY) VAL(0.5)

Example of a WRL triangle-mesh model:
 M21 = "Data/clocktwr.wrl" R(P(0 0 0) R20)



At the moment you can use the following file formats for external sub-models:
   GCAD, WRL, STL, OBJ.  You cannot use DXF, IGS, STP yet.
At the moment external sub-models cannot have internal or external sub-models.
 
 
 





M CatalogModel Pos [Vec/Refsys]

Positioning an Catalog-Part


CatalogPart
  - Select a catalogPart from list (or use middle-mousebutton)
  - or select an existing catalogPart

  The catalogPartname consists of catalog name, part name; see Catalog.


Position,Orientation
  Origin and orientation of the submodel

[Scale]:   magnifying scale; default is 1
  


Example external Model:
M22=CTLG "Schrauben/SKS_10x30" P20 VAL(1)

See also Functions for using catalog-parts.
