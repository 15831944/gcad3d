-------------------------------------------------------------------------
 Making of Vrml Models
-------------------------------------------------------------------------

 Load or make a Model.

 Export all surfaces of the model with the function:
 File / save Model as / VRML1 / SAVE
 All surfaces (also from subModels) are exported.
 Points and curves are not exported at present yet.
 Models, which are loaded already as Mockup models (vrml) can at present
 not be reExported. 



-------------------------------------------------------------------------
 Loading of Vrml Models
-------------------------------------------------------------------------

 Possibly delete Modelspace:
 File / new 

 Load several VRML1-Models:
 File / INSERT / select wrl-Model

 For each loaded VRML1-Modell a line similar to the following is generated:
 M1="SymDir/ModelName.wrl" P(0 0 0)
 (view with Dump / SOURCE or in the mode MAN)



-------------------------------------------------------------------------
 Positioning Vrml Models
------------------------------------------------------------------------- 

 Positioning with CAD:
   preselect S/M/DEL, select Model, Modify Button or right mouse button.
   Modify Position, Z and/or x axis.
   Preview with Tab-key or cursor a field more deeply.
   Store with Ok Button or right mouse button.
   Exit Button for leaving the Modify mode.
   Definiton of the axles:
     Default for Z-axis is DZ; (DX DY DIX DIY or D())
   The model can be positioned also by a reference system
     (a Plane - PLN - "R").

 CAD Analyze: For the selection of points with Mockup-models use
   Select / PT <-- Vertex


 Positioning with MAN:
   Modify the code line; Format:
   M # = "SymPfad/Modelname" position [ Scale ] [ z-vector [ x-vector ] ]
   Example:
   M20="Data/Niet1.dat" P(100 0) 0,5 DZ DIY
   The model is shifted 100,0 units, turned then by 90 degrees.
  
   A function for interactive positioning of subModels to a
   contact position is to be made.


-------------------------------------------------------------------------
 Store vrml assembly
-------------------------------------------------------------------------

 Store at present only in native format .dat:
 File / save Model as / native / key new fileName: e.g. Assy1.dat
